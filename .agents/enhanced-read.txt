# Task: Enhanced Health Data Types — HC Webhook Android App

## Context
You are working on the `health-connect-webhook` Android app at `~/Dev/Projects/health-connect-webhook`.
This app reads health data from Android Health Connect and POSTs it as JSON to configured webhook URLs.

## Current State
The app already supports these data types in `HealthConnectManager.kt`:
Steps, Sleep, HeartRate, HRV, Distance, ActiveCalories, TotalCalories, Weight, Height, BloodPressure, BloodGlucose, OxygenSaturation, BodyTemperature, RespiratoryRate, RestingHeartRate, Exercise, Hydration, Nutrition

## Your Task: Add Enhanced/Missing Data Types

Add support for these additional Health Connect data types that are NOT yet implemented:

1. **SpeedRecord** — instantaneous speed during exercise (m/s), expose as SpeedData with speed + time
2. **PowerRecord** — cycling/activity power in watts, expose as PowerData with watts + time  
3. **BodyFatRecord** — body fat percentage, expose as BodyFatData with percentage + time
4. **BoneMassRecord** — bone mass in kg, expose as BoneMassData with mass + time
5. **LeanBodyMassRecord** — lean body mass in kg, expose as LeanBodyMassData with mass + time
6. **MenstrualCycleRecord** (if API supports) — cycle data
7. **VO2MaxRecord** — VO2 max value (ml/kg/min), expose as Vo2MaxData with vo2Max + time
8. **FloorsClimbedRecord** — floors climbed count per interval

For each:
- Add enum entry in `HealthDataType`
- Add data class in `HealthConnectManager.kt`
- Add read function following existing patterns
- Wire into `readHealthData()` and `HealthData` data class
- Add permission in `ALL_PERMISSIONS` companion object
- Add serialization in `SyncManager.kt` JSON payload (follow existing pattern for each type)
- Add UI entry in `ConfigurationScreen.kt` data type toggles

## Constraints
- Follow existing code style (Kotlin, coroutines, result types)
- Use `ReadRecordsRequest` pattern for instantaneous records
- Use per-day `AggregateRequest` pattern for cumulative records (like steps/distance)
- Do NOT break existing functionality
- Build must pass: `./gradlew assembleDebug` from project root

## Deliverable
Working Kotlin code. Run `./gradlew assembleDebug` at the end and confirm build succeeds.
