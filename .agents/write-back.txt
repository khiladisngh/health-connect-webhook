# Task: Write-Back Feature — HC Webhook Android App

## Context
You are working on the `health-connect-webhook` Android app at `~/Dev/Projects/health-connect-webhook`.
This app reads health data from Android Health Connect and POSTs it to configured webhook URLs.

There is also a companion Python webhook server at `~/Dev/Projects/health-webhook/server.py` (runs on port 6655).
The server already has write-back endpoints:
- `GET /pending-writes` — returns pending write records `[{id, type, data, status, created_at}]`
- `POST /confirm-write` — confirms a write was committed (takes `{id}`)

## Your Task: Implement Write-Back (Server → Health Connect)

Add the ability for the app to periodically poll the server for pending records and write them INTO Health Connect.

### Supported write-back types (start with these):
1. **Nutrition** — `NutritionRecord` (calories, protein, carbs, fat, startTime, endTime)
2. **Hydration** — `HydrationRecord` (liters, startTime, endTime)
3. **Weight** — `WeightRecord` (kilograms, time)
4. **Steps** — `StepsRecord` (count, startTime, endTime) — manual entry

### Implementation plan:

**1. WriteBackManager.kt** (new file in `com.hcwebhook.app`):
- Polls configured webhook URLs for `/pending-writes` (GET request)
- Parses JSON response `{"pending": [{id, type, data, status}]}`
- For each pending record: writes to Health Connect using `HealthConnectClient.insertRecords()`
- Confirms each successful write by POSTing to `/confirm-write` with `{id}`
- Handles permissions — write permissions are separate from read permissions
- Returns `WriteBackResult` (success count, failure count, errors)

**2. Add write permissions** to `HealthConnectManager.kt`:
- Add `WRITE_PERMISSIONS` companion set with `HealthPermission.getWritePermission(...)` for:
  NutritionRecord, HydrationRecord, WeightRecord, StepsRecord
- Add `hasWritePermissions(): Boolean` method
- Add `insertNutrition(data)`, `insertHydration(data)`, `insertWeight(data)`, `insertSteps(data)` methods

**3. Wire into scheduled sync** — in `SyncWorker.kt` or `ScheduledSyncReceiver.kt`:
- After read sync completes, call `WriteBackManager.pollAndWrite()`
- Only if write-back is enabled in preferences

**4. Preferences** (`PreferencesManager.kt`):
- Add `KEY_WRITEBACK_ENABLED` boolean pref (default false)
- Add getter/setter `isWriteBackEnabled()` / `setWriteBackEnabled()`

**5. UI** — in `ConfigurationScreen.kt`:
- Add a toggle card "Write-Back (Server → Health Connect)" with explanation
- Only show if write permissions are granted

### Data format the server sends (in `data` field):
```json
// Nutrition
{"type": "nutrition", "data": {"calories": 500, "protein": 30, "carbs": 60, "fat": 15, "startTime": "2026-03-24T12:00:00Z", "endTime": "2026-03-24T12:30:00Z"}}

// Hydration  
{"type": "hydration", "data": {"liters": 0.5, "startTime": "2026-03-24T10:00:00Z", "endTime": "2026-03-24T10:05:00Z"}}

// Weight
{"type": "weight", "data": {"kilograms": 74.5, "time": "2026-03-24T08:00:00Z"}}

// Steps
{"type": "steps", "data": {"count": 1000, "startTime": "2026-03-24T09:00:00Z", "endTime": "2026-03-24T09:30:00Z"}}
```

## Constraints
- Follow existing Kotlin code style (coroutines, Result types, suspend functions)
- Do NOT break existing read/sync functionality
- Write permissions require separate Health Connect permission request — handle gracefully if not granted
- Build must pass: `./gradlew assembleDebug` from project root

## Deliverable
Working Kotlin code. Run `./gradlew assembleDebug` at the end and confirm build succeeds.
